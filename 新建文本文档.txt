# Toonflow 项目深度分析报告

## 1. 项目概览与架构设计

Toonflow 是一个基于 AI 的全流程短剧创作工具，采用 **Electron + Express + SQLite** 的技术架构。它通过多智能体协作（Multi-Agent Collaboration），实现了从小说文本到最终视频的自动化/半自动化生产流水线。

### 技术栈

* **后端/服务层**: Node.js, Express, Express-WS (WebSocket支持)

* **桌面容器**: Electron (支持跨平台打包)

* **数据库**: SQLite (使用 `better-sqlite3` 驱动, `knex` 作为 ORM/Query Builder)

* **AI 核心**: Vercel AI SDK (`ai` 包), 集成 DeepSeek, OpenAI, Anthropic, Gemini, Volcengine (豆包), Kling (可灵), Vidu 等多模型。

* **工具库**: `zod` (Schema验证), `sharp` (图片处理), `axios` (网络请求), `jsonwebtoken` (认证)。

### 目录结构核心

* `src/app.ts`: 后端服务入口，负责启动 Express 服务器，处理静态资源和路由。

* `src/agents/`: 智能体核心逻辑，包含大纲、剧本、分镜等 Agent 的编排。

* `src/routes/`: 业务路由控制器，按功能模块划分 (novel, outline, script, storyboard, video)。

* `src/utils/db.ts` & `src/lib/initDB.ts`: 数据库连接与初始化，定义了核心数据模型。

* `src/utils/ai/`: 统一的 AI 模型调用封装。

***

## 2. 核心业务流程 (Pipeline)

项目遵循线性创作流：**小说 -> 故事线 -> 大纲 -> 剧本 -> 分镜 -> 视频**

1. **小说导入 (Novel)**

   * 用户上传小说，系统解析章节存入 `t_novel` 表。
2. **大纲与故事线生成 (Outline & Storyline)**

   * **核心逻辑**: `src/agents/outlineScript/index.ts`

   * **协作模式**: 采用 "主程-子程" (Main-Sub Agent) 模式。

     * **Main Agent**: 协调员，负责调度子 Agent。

     * **Storyteller (AI1)**: 故事师，分析小说生成故事线 (`t_storyline`)。

     * **Outliner (AI2)**: 大纲师，根据故事线生成分集大纲 (`t_outline`)。

     * **Director**: 导演，负责审核 AI1 和 AI2 的输出，提出修改意见。
3. **剧本生成 (Script)**

   * 根据确定的分集大纲，生成详细的视觉剧本 (`t_script`)。
4. **资产生成 (Assets)**

   * 从大纲中提取角色、道具、场景信息，生成 AI 绘画提示词，并存入 `t_assets`。
5. **分镜制作 (Storyboard)**

   * **片段拆分**: `storyboard-segment` Agent 将剧本拆解为关键片段。

   * **镜头生成**: `storyboard-shot` Agent 为片段生成分镜提示词。

   * **绘图**: 调用绘图模型 (如 Midjourney, Stable Diffusion 接口) 生成分镜图 (`t_image`)。
6. **视频生成 (Video)**

   * 基于分镜图和提示词，调用视频生成模型 (如 Kling, Vidu) 生成最终视频片段 (`t_video`)。

## 3. 关键代码深入分析

### A. 数据库设计 (`src/lib/initDB.ts`)

这是理解业务数据的关键。

* **`t_prompts`**: 存储所有 Agent 的 System Prompt。这是系统的灵魂，定义了 Storyteller, Outliner, Director 等角色的行为准则。

* **`t_outline`**: 存储分集大纲，包含 `data` 字段 (JSON)，内含 `coreConflict` (核心矛盾), `emotionalCurve` (情绪曲线), `keyEvents` (关键事件) 等结构化数据。

* **`t_assets`**: 统一存储 角色、场景、道具，通过 `type` 字段区分。

* **`t_videoConfig`**: 存储视频生成的参数配置（分辨率、时长、模型参数）。

### B. 智能体编排 (`src/agents/outlineScript/index.ts`)

这是一个非常典型的 Agent 编排实现：

* **类结构**: `OutlineScript` 类封装了所有状态和逻辑。

* **工具定义 (Tools)**: 使用 `zod` 定义了 Agent 可调用的工具，如 `saveOutline`, `getChapter`, `AI1` (调用子 Agent)。

* **流式交互**: 使用 `u.ai.text.stream` 实现与 LLM 的流式对话，并通过 `EventEmitter` 将状态实时推送到前端。

* **Prompt 管理**: 动态从数据库 `t_prompts` 加载提示词，支持用户在运行时调整 Agent 行为。
